# Booster Rollups [An idea for scaling Ethereum applications without forcing every app to redeploy to every rollup.]

Booster rollups are a concept for letting L2s access Ethereum L1 state as if it were local enough to support a much more unified application environment.

In Taiko's roadmap, booster-rollup ideas inform the longer-term **Gwyneth** design.

## Why the Idea Matters

In a normal multi-rollup world, applications and liquidity tend to fragment:

- contracts get redeployed chain by chain
- state lives in isolated environments
- users and assets have to move around more often

Booster-rollup concepts aim to reduce that fragmentation by making L1 state and cross-rollup access more natural.

## Relationship to Taiko Gwyneth

Taiko Gwyneth adopts booster functionality as part of its horizontal scaling design. The goal is a network of Ethereum-equivalent rollups that feels more like one coordinated execution environment than a set of disconnected chains.

See [Taiko Gwyneth](/protocol/taiko-gwyneth) for the broader architecture.
