# Enable a Prover

Set up a Taiko prover to earn rewards by generating proofs.

## Overview

Taiko uses a multi-tier proof system. Each tier provides a different tradeoff between speed, cost, and security:

| Tier | Type | Speed | Bond | Security |
| --- | --- | --- | --- | --- |
| SGX | TEE attestation | Fast (seconds) | Lower (125 TAIKO) | Hardware-backed |
| ZK (RISC0, SP1) | Zero-knowledge proof | Slower (minutes) | Higher | Cryptographic |

Provers submit proofs for proposed blocks within a proving window. A **liveness bond** of at least 125 TAIKO (or TTKOh on testnet) is required. If the prover fails to submit a proof within the window, the bond is forfeited.

### Contestation

Any proof can be contested by submitting a higher-tier proof. If the original proof is found invalid, the original prover loses their bond and the contester receives a reward. This mechanism ensures proof integrity without requiring all blocks to use expensive ZK proofs.

## Prerequisites

- A running Taiko node (see [Run a Node](/guides/run-a-node))
- An [SGX v2 capable machine](https://github.com/ayeks/SGX-hardware#hardware-with-sgx2-support) (for SGX proving)
- TAIKO tokens (mainnet) or TTKOh tokens (Hoodi testnet) for the liveness bond
- An L1 account with ETH to pay for Prove Block transactions

## Set Up the Prover

::::steps

### Set up Raiko (SGX proof generator)

Raiko generates SGX proofs for Taiko blocks. Follow the Docker setup guide in the [Raiko repository](https://github.com/taikoxyz/raiko/blob/main/docs/README_Docker_and_RA.md).

### Configure the prover environment

Open your simple-taiko-node `.env` file and set these variables:

```bash
# Raiko endpoint from step 1
SGX_RAIKO_HOST=http://<your-raiko-host>:<port>

# Private key of the L1 account that will send Prove Block transactions
L1_PROVER_PRIVATE_KEY=<your-private-key>

# Enable the prover
ENABLE_PROVER=true

# Include prover in compose profiles
COMPOSE_PROFILES=l2_execution_engine,prover
```

:::warning
The account behind `L1_PROVER_PRIVATE_KEY` must have ETH on L1 to pay for proof submission transactions, and TAIKO (or TTKOh) approved for the liveness bond.
:::

### Approve token spending

The TaikoInbox contract must be approved to spend your TAIKO (or TTKOh) tokens for the liveness bond.

:::code-group

```txt [Mainnet]
1. Go to the TAIKO token contract on Etherscan:
   https://etherscan.io/address/0x10dea67478c5f8c5e2d90e5e9b26dbe60c54d800#writeProxyContract

2. Connect your wallet.

3. Call the `approve` function:
   - spender: 0x06a9Ab27c7e2255df1815E6CC0168d7755Feb19a (TaikoInbox)
   - amount: at least 125000000000000000000 (125 TAIKO with 18 decimals)

4. Submit the transaction.
```

```txt [Hoodi Testnet]
1. Go to the TTKOh token contract on Hoodi Etherscan:
   https://hoodi.etherscan.io/address/0xf3b83e226202ECf7E7bb2419a4C6e3eC99e963DA#writeProxyContract

2. Connect your wallet.

3. Call the `approve` function:
   - spender: 0xf6eA848c7d7aC83de84db45Ae28EAbf377fe0eF9 (TaikoInbox)
   - amount: at least 125000000000000000000 (125 TTKOh with 18 decimals)

4. Submit the transaction.
```

:::

:::info
If you already have TAIKO/TTKOh deposited in the TaikoInbox contract, those funds are used first before drawing from your allowance.
:::

### Start the prover

:::code-group

```bash [Mainnet]
docker compose up -d
```

```bash [Hoodi Testnet]
docker compose -f docker-compose-hoodi.yml up -d
```

:::

## Bonds and Economics

| Parameter | Value |
| --- | --- |
| Liveness bond | 125 TAIKO (mainnet) / 125 TTKOh (Hoodi) |
| Bond token | TAIKO on Ethereum mainnet / TTKOh on Hoodi |
| Bond return | Returned after successful proof submission |
| Bond forfeiture | Forfeited if proof is not submitted within the proving window |

The liveness bond ensures provers are accountable. When you are assigned a block:

1. The bond amount is locked from your TaikoInbox deposit (or allowance).
2. Submit a valid proof within the proving window to reclaim your bond plus the proving reward.
3. If you miss the window, the bond is forfeited and another prover can prove the block.

## Monitoring

Check your prover logs:

```bash
docker compose logs -f taiko_client_prover_relayer
```

Search for a specific block assignment:

```bash
docker compose logs taiko_client_prover_relayer | grep "<block-number>"
```

## Troubleshooting

| Problem | Cause | Fix |
| --- | --- | --- |
| Block proposed but no proof submitted | Raiko failed or proof request was not sent | Set `PROVER_STARTING_BLOCK_ID=<block-number>` in `.env`, then restart with `docker compose down && docker compose up -d` (do **not** use `-v`) |
| `Start: Create EVM proof; Killed` | Insufficient RAM | Ensure your machine meets the minimum 16 GB RAM requirement |
| Proof window expired | Proof generation too slow or prover offline | Check Raiko logs for errors; ensure your SGX hardware is functioning correctly |
| Bond forfeited | Proof not submitted in time | Monitor assignments closely; consider running redundant provers |

:::warning
When restarting the prover to retry a block, never use `docker compose down -v`. The `-v` flag deletes volumes and forces a full resync from genesis.
:::
