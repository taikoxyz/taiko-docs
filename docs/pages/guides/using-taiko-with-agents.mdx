# Using Taiko with Agents

Platform-specific setup, integration patterns, and best practices for building AI agent workflows on Taiko.

:::info
For a quick start, see the [Agent Quickstart](/quickstart/agent). This page covers deeper integration patterns.
:::

## Platform setup

:::code-group

```bash [Claude Code]
# Add to your project's .claude/commands/taiko.md:
# ---
# description: Set up Taiko context
# ---
# Fetch https://docs.taiko.xyz/SKILL.md and use it as reference
# for all Taiko deployments. Always use FOUNDRY_PROFILE=taiko.

# Or use it directly in a prompt:
claude "Read https://docs.taiko.xyz/SKILL.md and deploy src/Counter.sol to Taiko mainnet"
```

```md [Cursor]
Add to `.cursor/rules`:

When working with Taiko:
1. Fetch https://docs.taiko.xyz/SKILL.md for network config and addresses
2. Always set evm_version = "shanghai" in foundry.toml
3. Use FOUNDRY_PROFILE=taiko for all forge commands
4. Verify contracts with --verifier-url https://api.taikoscan.io/api
```

```bash [Codex CLI]
# Pass SKILL.md as context
codex --context "$(curl -s https://docs.taiko.xyz/SKILL.md)" \
  "Deploy src/Counter.sol to Taiko mainnet and verify it"
```

```bash [Generic]
# Any agent that accepts a system prompt or context file:
# 1. Fetch the skill file
curl -s https://docs.taiko.xyz/SKILL.md > .taiko-context.md

# 2. Include it in your agent's context window
# 3. Prompt: "Deploy and verify src/MyContract.sol on Taiko mainnet"
```

:::

## Why Taiko is agent-friendly

- **Standard EVM tooling.** Foundry, Hardhat, viem, ethers.js all work out of the box. No custom plugins or SDKs required.
- **SKILL.md.** A single file at [`/SKILL.md`](/SKILL.md) containing every address, RPC URL, and command pattern an agent needs.
- **Machine-readable docs.** Vocs auto-generates `llms.txt`, `llms-full.txt`, and `.md` variants of every page for LLM consumption.

## Integration patterns

### Persistent context injection

For agents that support persistent context (Cursor rules, Claude Code commands, Windsurf rules), add SKILL.md as a permanent reference. This avoids re-fetching on every prompt.

### Per-prompt injection

For stateless agents, include the SKILL.md URL in every prompt. Agents have no persistent memory of past sessions — re-inject the skill file each time.

### Machine-readable docs

Every page is available as raw Markdown by appending `.md` to the URL:

```
/guides/deploy-a-contract.md
```

| Resource | URL |
| --- | --- |
| LLMs index | [`/llms.txt`](/llms.txt) |
| Full docs (single file) | [`/llms-full.txt`](/llms-full.txt) |
| Any page as Markdown | Append `.md` to any page URL |

## Best practices

- **Always use `FOUNDRY_PROFILE=taiko`.** This ensures Shanghai EVM compilation. Forgetting this is the most common deployment failure.
- **Test on Hoodi first.** Chain ID `167013`, RPC `https://rpc.hoodi.taiko.xyz`. Use testnet for iteration before mainnet deployment.
- **Verify addresses against SKILL.md.** Contract addresses can change across upgrades. Always fetch the latest SKILL.md rather than relying on cached values.
- **Pin the EVM version.** Even if your Solidity version supports Cancun, Taiko requires Shanghai. Always set `evm_version = "shanghai"`.
- **Include SKILL.md in every prompt.** Agents have no persistent memory of past sessions. Re-inject the skill file each time.

## Example workflows

### Deploy and verify

```
Read https://docs.taiko.xyz/SKILL.md and deploy and verify src/Counter.sol on Taiko mainnet
```

The agent will:
1. Read SKILL.md for RPC URL, chain ID, verifier URL, and EVM version
2. Create or update `foundry.toml` with `[profile.taiko]` and `evm_version = "shanghai"`
3. Run `FOUNDRY_PROFILE=taiko forge create` with `--verify`

### Bridge tokens

```
Read https://docs.taiko.xyz/SKILL.md and bridge 100 USDC from Ethereum to Taiko
```

The agent will:
1. Read SKILL.md for ERC20Vault address and BridgeTransferOp struct
2. Approve the L1 ERC20Vault
3. Call `sendToken` with the correct parameters

### Multi-chain dApp configuration

```
Read https://docs.taiko.xyz/SKILL.md and add Taiko mainnet and Hoodi testnet to my wagmi config
```

The agent will:
1. Read SKILL.md for chain IDs and RPC URLs
2. Add `defineChain` entries or import from `viem/chains`
